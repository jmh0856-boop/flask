# ORM과 SQLAlchemy

## ORM의 의미, 사용 이유
- **의미**: ORM(Object-Relational Mapping, 객체-관계 매핑)은 객체 지향 프로그래밍 언어의 객체(Object)와 관계형 데이터베이스의 데이터(Relational)를 자동으로 연결(Mapping)하는 프로그래밍 기술 입니다.
  - 객체(Object) - Python (Flask, Django)
  - 관계형(Relational)데이터베이스 DB - RDBMS
  - 위 2개를 Mapping(연결) 시켜주는 것
- 객체: 객체, 클래스, 속성의 구조
- 관계형 데이터베이스: 테이블, 로우 칼럽과 같은 구조

- **사용 이유**: SQL 쿼리문을 직접 작성할 필요 없이 파이썬 코드만으로 데이터베이스 조작이 가능해져, 개발 생산성이 향상되고 유지보수가 쉬워집니다. 또한, 데이터베이스 종류가 바뀌어도 코드 변경을 최소화할 수 있습니다.
 
## Model 정의하기
- SQLAlchemy에서 모델(Model)은 데이터베이스 테이블에 매핑되는 파이썬 클래스를 의미합니다. declarative_base()를 상속받아 클래스를 선언하고, Column, Integer, String 등의 속성을 사용하여 테이블의 필드(컬럼)와 제약 조건을 정의합니다.

## 모델 간의 관계 해석
- 데이터베이스의 외래 키(Foreign Key) 관계를 ORM 모델에서는 클래스 간의 관계로 표현합니다. 예를 들어, User와 Todo 모델이 있다면, 사용자가 여러 할일을 가질 수 있도록 relational() 함수 등을 이용해 객체지향적인 방식으로 관계를 설정하고 해석합니다.

## 스키마와 모델 사이의 차이점
- **모델**: 애플리케이션 내에서 데이터를 다루는 프로그래밍 언어 수준의 추상적인 데이터 구조(파이썬 클래스)입니다.
- **스키마**: 실제 데이터베이스에 데이터가 어떻게 저장될지 정의하는 구체적인 구조(테이블, 컬럼, 제약 조건 등)를 의미합니다.
- **ORM 역할**: ORM은 모델 정의를 바탕으로 실제 데이터베이스 스키마를 생성하고, 둘 사이의 데이터 변환을 담당합니다.

## CRUD 테스팅
- CRUD는 Create(생성), Read(읽기), Update(수정), Delete(삭제)의 약자로, 데이터베이스의 기본적인 네 가지 작업을 말합니다. SQLAlchemy에서는 session.add(), session.query().all, session.commit(), session.delete() 등의 메서드를 사용하여 이러한 작업을 테스트하고 수행합니다.

## 직렬화와 역직렬화의 의미
- **직렬화(Serialization)**: 파이썬 객체나 데이터를 네트워크를 통해 전송하거나 파일로 저장하기 위해 바이트 스트림이나 JSON 문자열 같은 연속된 형태로 변환하는 과정입니다.
- **역직렬화(Deserialization)**: 반대로 바이트 스트림이나 JSON 문자열 형태의 데이터를 다시 원본ㅌ 파이썬 객체로 복원하는 과정입니다.
- **사용 이유**: 메모리에 상주하는 객체는 다른 시스템에서 직접 사용할 수 없으므로, 데이터 교환 시 이 과정이 필수적입니다.
